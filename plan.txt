The relay manages sessions, validates protocol messages, and routes commands and state updates between PCs and remotes.

- When a device (socket) tries to register with a new `sid`, it is first removed from any previous session it was part of.
- The `sessionManager` should track which socket is associated with which `sid`, and prevent double registration.

#### Where to enforce this

- In `sessionManager.registerPc` and `sessionManager.registerRemote`, before registering a socket to a new `sid`, check if it is already registered to another session.
- If so, remove it from the previous session before adding it to the new one.

#### Example logic (pseudo-code):

```js
// In sessionManager.js

function registerPc(sid, ws) {
  // Check if ws is already registered to another sid
  const oldMeta = getSocketMeta(ws);

  }
  // Proceed with registration as usual...
}
```
